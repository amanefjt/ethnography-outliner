# タスク：ビジュアルの強化と表示バグの修正

カード・グループの配色ルールを変更し、選択時のスタイルを強調するとともに、グループ名の表示位置の不具合を修正する。

## 目的
- 色変更時に縁だけでなく、カード全体の背景色が変わるようにして、視認性を高める。
- 選択されたアイテム（ノート・グループ）の枠線を太くし、どれが選択されているか明確にする。
- グループをドラッグした際に名前部分（ヘッダー）がズレる、または変な位置に表示されるバグを修正する。

## 完了定義 (DoD)
- [ ] 各カラークラス（slate, blue, green, yellow, rose）において、背景色と枠線色が調和したデザインが適用されている。
- [ ] 選択されたカード・グループの枠線が 3px 程度に太くなり、一目で選択中だとわかる。
- [ ] グループをドラッグしている最中も、グループ名（ヘッダー）がカードの上端に正しく固定されている。
- [ ] 折りたたみ時もデザインが崩れず、一貫した配色が適用されている。

## ステップ
1. [ ] `style.css`: カラーバリエーション（背景色、枠線色、テキスト色）を CSS 変数として整理。
2. [ ] `style.css`: `.selected` クラスの `border-width` を強化。
3. [ ] `style.css`: `.group-header` の配置を `sticky` から `absolute` もしくは安定した `relative` 配置に変更し、ドラッグ時の挙動を安定化。
4. [ ] `main.js`: 必要に応じて HTML 構造の微調整。
5. [ ] 動作確認。
