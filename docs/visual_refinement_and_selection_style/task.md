# タスク：ビジュアルの洗練と選択時スタイルの改善

選択時の視認性向上、配色ルールの動的化、およびUI要素（ID表示、ボタン配置）の微調整を行う。

## 目的
- グループを選択した際も中のカードが隠れないようにし、編集作業をスムーズにする。
- 選択時の枠線色を、各アイテムのベースカラーに合わせた強調色に変更する（オレンジ固定を廃止）。
- 折りたたみ（マスク）時のボタン配置を「左上」に固定し、操作の一貫性を保つ。
- ID表示をさらに控えめにし、不要な区切り線を修正する。

## 完了定義 (DoD)
- [ ] グループを選択した際、背景が中身を覆い隠さず、中のカードが正しく表示される。
- [ ] 青いカードを選択すると濃い青の枠、緑なら濃い緑の枠というように、ベースカラーに応じた選択色が表示される。
- [ ] 折りたたみ時のボタンが、中央ではなくカードの左上に配置されている。
- [ ] ID表示部分の境界線（border-top）が消え、フォントサイズがさらに小さくなっている。

## ステップ
1. [ ] `style.css`: 各カラークラス（.color-blue 等）に対して、対応する `--selection-color` を定義。
2. [ ] `style.css`: `.selected` のスタイルを汎用的なオレンジ固定から、変数 `--selection-color` 使用に変更。
3. [ ] `style.css`: `.group-card.selected` の背景色が中身を隠していないか確認し、透過度を調整。
4. [ ] `style.css`: `.collapsed` 時のボタン位置を `top: 8px; left: 8px;` 等の左上配置に修正（中央揃えの干渉を排除）。
5. [ ] `style.css`: `.meta` クラスの `border-top` 削除と `font-size` 調整。
6. [ ] 動作確認。
